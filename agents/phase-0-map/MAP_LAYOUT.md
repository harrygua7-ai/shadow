# MAP_LAYOUT

## 文件目的

本文档描述 Shadow MVP 外部小镇地图的布局方案。

它基于 Phase 0 概念图，用于指导后续 Godot TileMap 实现。

## 地图总体结构

第一版地图为中等规模外部小镇地图。

整体结构：

> 公共广场位于地图中心，主要地点围绕广场分布，主路连接各区域，地图边缘保留未来扩展道路。

地图不是单一街道，也不是完整大世界，而是一个紧凑但可探索的小镇外部空间。

## 推荐地点布局

### 中心：公共广场

位置：

```text
地图中心
```

作用：

- 小镇公共中心
- 未来公告、事件、集会入口
- 连接各地点的道路节点

设计：

- 保持开阔
- 可放公告板或中心标志
- 周围道路向四周延伸
- 不要堆太多树和装饰

### 左上：政府大厅

位置：

```text
地图左上区域
```

原因：

- 给公共权力和秩序一个稳定、正式的位置
- 与广场有清晰道路连接
- 不与酒馆/市场过度混杂

设计：

- 建筑适中
- 有台阶和前庭
- 周围可有少量路灯和旗帜
- 不要做成巨大建筑

### 中上：酒馆

位置：

```text
地图中上偏左或中上区域
```

原因：

- 接近广场，方便成为社交入口
- 与小巷相邻，方便传闻/秘密玩法
- 与市场有自然关系

设计：

- 建筑适中
- 门口清晰
- 可有少量户外桌椅
- 不阻塞主路

### 酒馆右侧：小巷

位置：

```text
酒馆和市场之间
```

原因：

- 形成从社交场到交易场之间的灰色通道
- 适合未来秘密、传闻、打听消息

设计：

- 狭窄
- 阴影更重
- 两侧有建筑墙、箱子、侧门
- 中间通道必须可走
- 不要被杂物堵死

### 右上：市场 / 商店街

位置：

```text
地图右上区域
```

原因：

- 与小巷、酒馆形成交易和传闻流动
- 远离政府大厅，形成社会功能区差异

设计：

- 多个小摊和店面
- 中央通道宽
- 摊位靠边
- 保持可移动空间

### 左下：档案馆

位置：

```text
地图左下区域
```

原因：

- 与政府大厅同属秩序/记忆相关区域，但更安静
- 离酒馆和市场稍远，形成节奏差异

设计：

- 安静正式
- 有石路、少量树木、书馆感
- 门口明确
- 不要被树挡住

### 右下：工坊

位置：

```text
地图右下区域
```

原因：

- 与市场形成生产-交易关系
- 放在边缘更合理，适合烟囱、材料堆、工作台

设计：

- 有材料、工具、工位
- 中央留通道
- 材料堆靠边
- 不影响主路

### 边缘：未来扩展道路

位置：

```text
地图边缘多个方向
```

作用：

- 暗示小镇未来可以扩张
- 后续连接新区域、社区、品牌/组织据点

设计：

- 可以有路障
- 可以有断桥
- 可以有施工感
- 不需要现在可进入

## 道路结构

道路应连接：

```text
政府大厅 <-> 广场
酒馆 <-> 广场
酒馆 <-> 小巷 <-> 市场
市场 <-> 广场
档案馆 <-> 广场
工坊 <-> 广场
工坊 <-> 市场
广场 <-> 未来扩展道路
```

道路设计原则：

- 主路宽
- 小巷窄
- 广场开阔
- 门口区域清楚
- 不用树和装饰堵主路

## 建筑尺寸原则

建筑应当作为地点入口，而不是完整室内空间。

建议：

- 政府大厅可略大，但不能压迫地图
- 酒馆中等偏小
- 市场摊位小而多
- 档案馆中等
- 工坊横向可稍宽，但内部物件靠边
- 所有建筑周围必须留可走空间

## 可走空间原则

外部地图应支持未来 WASD 移动。

因此：

- 主路至少要有明显宽度
- 小巷可以窄，但不能只有一格
- 广场要能容纳玩家、NPC、未来事件标记
- 门口前要有停留空间
- 树、灯、摊位不要连续堵路

## 初步 Tile 分类

### 地面类

- 石路
- 广场石砖
- 草地
- 台阶
- 小巷暗色地砖
- 桥面/边缘道路

### 建筑类

- 墙体
- 屋顶
- 门
- 窗户
- 台阶
- 招牌底座

### 装饰类

- 树
- 路灯
- 长椅
- 公告板
- 箱子
- 桶
- 摊位
- 工坊材料
- 栅栏

### 边界类

- 水体
- 岩石边缘
- 路障
- 封锁道路

## 碰撞布局原则

不可走：

- 建筑主体
- 树干和树冠占位
- 水体
- 路障
- 墙
- 摊位主体
- 工坊大型材料堆
- 路灯底座

可走：

- 主路
- 广场
- 门口台阶
- 小巷中心通道
- 摊位前方
- 建筑前庭

## 门口预留

必须预留以下门口：

- 政府大厅门口
- 酒馆门口
- 市场主入口/摊位街入口
- 档案馆门口
- 工坊门口
- 小巷入口

Phase 0-2 不做入口逻辑，但 Phase 4 后可在这些位置添加提示和场景切换。

## 昼夜预留

地图中应预留：

- 路灯位置
- 建筑窗户
- 酒馆暖光点
- 小巷暗区
- 市场灯串/摊位灯
- 政府大厅公共灯
- 工坊火光/炉光

Phase 0-2 不实现昼夜，只保证地图结构支持后续添加。

## 当前风险点

1. 建筑过大会压缩道路。
2. 树和装饰过多会影响未来移动。
3. 小巷如果太窄会导致碰撞手感差。
4. 市场摊位如果太密，会像障碍迷宫。
5. 概念图不能直接等同 TileMap，需要工程化简化。
6. 地点名不能画死，否则后续难改。
7. 如果道路不够宽，Phase 3 移动手感会很差。

## Phase 1 实现建议

Phase 1 应先做可用原型美术，而不是纯色白盒。

建议实现顺序：

1. 创建 `scenes/world/town_exterior.tscn`
2. 创建基础 tileset
3. 先铺地面和道路
4. 放建筑体块
5. 放门口和台阶
6. 放少量树和灯
7. 放市场、工坊装饰
8. 保留扩展道路
9. 检查可走空间是否足够

## Phase 2 实现建议

Phase 2 在 Phase 1 地图基础上添加碰撞。

优先碰撞对象：

1. 建筑主体
2. 水体/边界
3. 墙和栅栏
4. 树
5. 摊位主体
6. 工坊材料堆
7. 路灯底座

Phase 3 移动引擎接入后再微调碰撞体大小。
